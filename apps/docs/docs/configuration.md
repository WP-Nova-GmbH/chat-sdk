All options are passed to `WpNova("init", config)` or the `init(config)` npm helper. The only required fields are `publicSurfaceId` and `tokenEndpoint`.

```ts
import { init } from "@wp-nova/sdk";

init({
  publicSurfaceId: "surf_...",
  tokenEndpoint: "/api/nova-token",
  baseUrl: "https://chat.wp-nova.ai",
  safeValueSelectors: ["#case-number", ".agent-safe-field"],
});
```

## Options

| Field | Required | Description |
| --- | --- | --- |
| `publicSurfaceId` | Yes | Non-secret surface handle generated by Nova admin, for example `surf_...`. This is the only surface identifier that belongs in browser code. |
| `tokenEndpoint` | Yes | Your backend route. The SDK posts `{ publicSurfaceId, origin }` to this endpoint with `credentials: "include"`. |
| `baseUrl` | No | Nova iframe host. Defaults to `https://chat.wp-nova.ai`. Local development commonly uses `http://localhost:5173`. |
| `mount` | No | CSS selector or `HTMLElement` to mount into. Defaults to `document.body`. |
| `title` | No | Pre-auth launcher/panel title shown before trusted surface display settings arrive. |
| `accent` | No | Pre-auth accent color for the SDK-owned launcher shell. |
| `triggerColor` | No | Launcher button color. Defaults to `accent`. |
| `triggerIconColor` | No | `light`, `dark`, or a custom hex color for the launcher icon. |
| `safeValueSelectors` | No | CSS selectors that opt field values into page snapshot capture. Field values still pass sensitivity checks. |
| `voiceMode` | No | Enables the embedded voice button and delegates microphone access to the Nova iframe. Defaults to `false`. |
| `protocolVersion` | No | Bridge protocol override for compatibility testing. Do not set in normal integrations. |

## Defaults

```ts
init({
  publicSurfaceId: "surf_...",
  tokenEndpoint: "/api/nova-token",
  baseUrl: "https://chat.wp-nova.ai",
  title: "Assistant",
  accent: "#8665e3",
  triggerIconColor: "light",
});
```

If `accent` or `triggerColor` is not supplied, the SDK can wait for trusted surface theme data before showing the launcher. Supplying one of those colors gives a branded first paint before authentication completes.

## Token Endpoint Request

The SDK request body is intentionally small:

```json
{
  "publicSurfaceId": "surf_...",
  "origin": "https://app.example.com"
}
```

The SDK includes cookies with `credentials: "include"` so your backend can authenticate the current user. Do not add `email`, `userId`, or the integration secret to the browser config or request body.

Your backend should call Nova with the authenticated user's email and pass through either:

```json
{ "access_token": "<embedded-session token>", "expires_in": 900 }
```

or:

```json
{ "unavailable": true, "email": "person@example.com", "message": "No Nova account found." }
```

## Re-initialization

The SDK is singleton-safe. Re-running `init` during HMR, route-level remounts, or duplicate script loads reuses the existing custom element.

If `publicSurfaceId`, `baseUrl`, or `protocolVersion` changes, the element rebuilds the iframe and bridge, clears buffered auth, and fetches a fresh token before posting auth to the iframe.

## Destroying the Embed

Use `destroy()` or `WpNova("destroy")` only when your app truly disables or removes the assistant. Normal SPA route changes do not require unmounting.

```ts
import { destroy } from "@wp-nova/sdk";

destroy();
```
