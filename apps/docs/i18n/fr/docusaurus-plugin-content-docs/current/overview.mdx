## Vue d’ensemble

Le Nova Chat SDK est le package navigateur qui monte une iframe de chat hébergée par Nova dans une page appartenant au client. Il est volontairement léger : le SDK gère le lanceur, le conteneur iframe, la transmission du token et le bridge `postMessage`. L’iframe gère l’interface de conversation et communique avec les services Nova au moyen d’un token de session intégrée à courte durée de vie.

Le navigateur ne reçoit jamais de secret d’intégration. Votre app expose un `tokenEndpoint` qui authentifie l’utilisateur courant, appelle `POST /embed/session` de Nova depuis le serveur, puis renvoie la réponse de session intégrée au SDK.

Le bridge du SDK gère trois capacités de la page hôte :

- Des instantanés de page qui résument le texte visible, les liens, les contrôles, les libellés, la sélection et les valeurs de champ sûres.
- Des actions de navigation intégrées sur des handles d’éléments stables issus du dernier instantané.
- Des outils d’intégration enregistrés par votre app avec `registerToolHandler`.

## Packages

| Package | Rôle |
| --- | --- |
| `@wp-nova/sdk` | SDK navigateur indépendant du framework et artefact CDN. |
| `@wp-nova/sdk-react` | Provider, composant et hooks React autour du SDK cœur. |
| `@wp-nova/sdk-angular` | Provider, service et composant standalone Angular autour du SDK cœur. |

## Modèle d’exécution

1. La page cliente charge `@wp-nova/sdk` par balise script ou via npm.
2. La page appelle `WpNova("init", { publicSurfaceId, tokenEndpoint })`.
3. Le SDK monte `<wp-nova-chat>`, pointe son iframe vers `<baseUrl>/embed/chat?surface=<publicSurfaceId>` et récupère un token depuis le backend client.
4. L’iframe envoie des requêtes de bridge comme `REQUEST_SNAPSHOT` ou `CLIENT_TOOL_REQUEST`.
5. Le SDK ne répond qu’après des contrôles stricts de l’origine et de la source.
