## Übersicht

Das Nova Chat SDK ist das Browser-Paket, das ein von Nova gehostetes Chat-iframe in eine kundeneigene Seite einbindet. Es ist bewusst schlank: Das SDK steuert den Launcher, den iframe-Container, die Token-Übergabe und die `postMessage`-Bridge. Das iframe steuert die Konversationsoberfläche und kommuniziert mit Nova-Diensten über ein kurzlebiges Embedded-Session-Token.

Der Browser erhält niemals ein Integrations-Secret. Deine App stellt einen `tokenEndpoint` bereit, der den aktuellen Benutzer authentifiziert, Nova serverseitig über `POST /embed/session` aufruft und die Embedded-Session-Antwort an das SDK zurückgibt.

Die SDK-Bridge verarbeitet drei Fähigkeiten der Host-Seite:

- Seiten-Snapshots, die sichtbaren Text, Links, Bedienelemente, Labels, Auswahl und sichere Feldwerte zusammenfassen.
- Eingebaute Navigationsaktionen gegen stabile Element-Handles aus dem neuesten Snapshot.
- Integrator-Tools, die deine App mit `registerToolHandler` registriert.

| `@wp-nova/sdk-angular` | Angular-Provider, Service und Standalone-Komponente rund um das Core-SDK. |

## Laufzeitmodell

1. Die Kundenseite lädt `@wp-nova/sdk` per Script-Tag oder npm.
2. Die Seite ruft `WpNova("init", { publicSurfaceId, tokenEndpoint })` auf.
3. Das SDK mountet `<wp-nova-chat>`, zeigt das iframe auf `<baseUrl>/embed/chat?surface=<publicSurfaceId>` und holt ein Token vom Kunden-Backend.
4. Das iframe sendet Bridge-Anfragen wie `REQUEST_SNAPSHOT` oder `CLIENT_TOOL_REQUEST`.
5. Das SDK antwortet nur nach strikten Origin- und Source-Prüfungen.
